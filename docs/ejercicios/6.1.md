# Ejercicio 6.1 — Tipos de herencia, clases y subclases (RA7.a)

Basado en la teoría de jerarquía de clases (Unidad 6.1).

## Objetivo

Crear **dos** estructuras de herencia (elige **2** de estas 4): **Especialización**, **Extensión**, **Especificación**, **Construcción**.

> Nota: el comportamiento no tiene que ser “realista”, pero las clases deben diferenciarse claramente para que se entienda el **tipo de herencia**.

## I. Práctica

Sigue los pasos para cada una de las 2 estructuras elegidas (Especialización, Extensión, Especificación, Construcción). Usa Enlaces permanentes a código para ilustrar tus respuestas:

I.a) Para **cada una** de las 2 estructuras elegidas:

- Piensa en **(1) una superclase** y **(2) dos subclases**.
- Crea las clases en Kotlin.

RESPUESTA: Explica tu propuesta de superclase y subclases para cada estructura, y el tipo de herencia que representa.

Primera propuesta (Especialización):
He pensado en una clase padre `Empleado` que tiene un nombre, un salario y sabe `trabajar()`. Luego he hecho dos hijas: `Desarrollador` y `Gerente`. Es herencia por Especialización porque las hijas tienen el mismo comportamiento que el padre (las dos trabajan y calculan sueldo) pero lo hacen de forma más específica o detallada, por eso hacemos un `override` de sus métodos.

Segunda propuesta (Extensión):
He hecho una clase padre `Dispositivo` que sirve para encender y apagar. Luego hice el `TelefonoInteligente` y `RelojInteligente` que heredan de él. Esto es herencia por Extensión, porque además de encender y apagar, los hijos "añaden" funciones nuevas que el padre no tenía, como `llamar()` o `medirRitmoCardiaco()`.

I.b) Crea un ejemplo de uso con `main`:

- Asigna a una variable de la **superclase** instancias de las **subclases** (polimorfismo).
- Muestra salida por consola con “logs” (`println`) para que se vea el comportamiento y el flujo del programa.

RESPUESTA: Explica tu propuesta de ejemplo de uso, qué se imprime en cada caso y por qué.

Primer ejemplo (Especialización):
He creado una lista de tipo `Empleado` metiendo dentro tanto empleados normales, como desarrolladores y gerentes. Con un bucle `for` llamo al método `trabajar()`. Lo interesante es que aunque todos son `Empleado`, como Kotlin usa polimorfismo, llama al código de la clase hija y por eso se imprime el texto propio de cada trabajador.

Segundo ejemplo (Extensión):
Tengo otra lista pero de `Dispositivo`. Puedo encender y apagar todos usando un bucle. Sin embargo, para poder usar los métodos nuevos ("extendidos") tengo que usar un `when (dispositivo)` para comprobar de qué clase es exactamente. Si es un teléfono llama, y si es un reloj mide el pulso.
